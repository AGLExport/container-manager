root base autoboot
Type: boolean Default: false

When this property is true, this guest container launch at boot time.

root base bootpriority
Type: integer Default: 1000

Container managers launch guest container in order of priority in boot time. 0: Highest. 9999: Lowest.

Value must be greater or equal to 0 and lesser or equal to 9999

Must contain a maximum of 2 items

root fs mount A mount item.
Type: object

root fs mount A mount item. type
Type: enum (of string)

When you want to mount tmpfs, this property shall be set filesystem. When you want to mount host directory to guest directory, this property shall be set directory.

Must be one of:

  • "filesystem"
  • "directory"

root fs mount A mount item. from
Type: string

When you want to mount tmpfs, this property shall be set file system name. When you want to mount host directory to guest directory, this property shall be set host side directory.

root fs mount A mount item. to
Type: string

This property shall be set guest side directory without top /.

Must match regular expression: ^(dev|sys|etc|run|var|opt)
Examples:

"dev/shm"

"run"

"sys/kernel/security"

"run/udev/data"

"var/display/drm-lease-manager/lease"

"var/pipewire-0"

"etc/systemd/system.conf.d/10-environment.conf"

root fs mount A mount item. fstype
Type: string

When you want to mount tmpfs, this property shall be set tmpfs. When you want to mount host directory to guest directory, this property shall be set none.

root fs mount A mount item. option
Type: string

The property set mount options for each mount operations. More detail of mount options refer to example.

root device static A static device item. type
Type: enum (of string)

A devnode use to one by one device file setting. ex. /dev/ttyS0. A devdir use to device file directory based setting. ex. /dev/dri. A gpio use to sysfs gpio. A iio use to sysfs iio with device file.

Must be one of:

  • "devdir"
  • "devnode"
  • "gpio"
  • "iio"

root device static A static device item. from
Type: string

This property is set device file path in host in case of devnode and iio. This property is set device file directory in host in case of devdir. This property is set sysfs path in host in case of gpio.

root device static A static device item. to
Type: string

This property is set device file path in guest in case of devnode and iio. This property is set device file directory in guest in case of devdir. This property is set sysfs path in guest in case of gpio.

root device static A static device item. devnode
Type: string

This property is set representative device file path in guest in case of devnode, devdir and iio. This property use to automatic major/minor number scanning to allow device access. In case of gpio, this property is not necessary.

root device static A static device item. optional
Type: integer

When this property set 0, if container manager can't access this device, container manager will not launch this guest container. When this property set 1, if container manager can't access this device, container manager will disable this device and launch this guest container.

Value must be greater or equal to 0 and lesser or equal to 1

root device static A static device item. wideallow
Type: integer

When this property set 0, container manager set allow device access major: minor number. When this property set 0, container manager set allow device access major: all minor number.

Value must be greater or equal to 0 and lesser or equal to 1
